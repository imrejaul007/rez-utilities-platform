# SCHEMA-001-059: Backend Models & Schema Audit

**59 unique schema/model issues across 9 services**
**Services:** rez-backend-master, rez-payment-service, rez-wallet-service, rez-order-service, rez-merchant-service, rez-catalog-service, rez-karma-service, rez-finance-service, rez-shared
**Audit Source:** Backend Models Schema Audit Agent
**Scope:** 200+ model files analyzed

---

## CRITICAL (10)

### SCHEMA-001: `strict: false` on GroupBuy, MerchantStaff, AdminBroadcast Models

`strict: false` completely disables Mongoose validation — any field can be written to the document regardless of schema definition.

---

### SCHEMA-002: `Math.random()` for Code Generation — 3 Models

**GroupBuy** (`inviteCode`): `Math.random()` generates predictable invite codes
**GiftCard** (`code`): Gift card codes are predictable
**Consultation** (`consultationNumber`): Consultation numbers are predictable

Fix: Use `crypto.randomBytes(16).toString('hex')`

---

### SCHEMA-003: FlashSale Index References Non-Existent Field `isActive`

**File:** `FlashSale.ts:284`

Index defined on `isActive` field, but the field is named `enabled`. The index is on a non-existent field.

**Method at line 318** also references `this.isActive` when the field is named `enabled`.

---

### SCHEMA-004: TableSession Status Enum Mismatch

**File:** `TableSession.ts`

Interface defines 5 status values but schema only has 3. Type mismatch between TypeScript interface and Mongoose schema.

---

### SCHEMA-005: BlockedSlot `merchantId` References Wrong Model

**File:** `BlockedSlot.ts:25`

```typescript
merchantId: { type: Schema.Types.ObjectId, ref: 'User' }  // Should be 'Merchant'
```

Points to User instead of Merchant model.

---

### SCHEMA-006: Merchant Model Has 13 `Schema.Types.Mixed` Fields

**File:** Merchant model

13 instances of `Schema.Types.Mixed` including sensitive credential storage. Untyped fields prevent validation and cause silent data drift.

---

### SCHEMA-007: AES-256-CBC vs AES-256-GCM Inconsistency

**File:** `rez-wallet-service/MerchantWallet.ts`

Uses AES-256-CBC for encryption. Backend model uses AES-256-GCM. Cross-service data cannot be decrypted.

---

### SCHEMA-008: MerchantWallet Encryption Silently Falls Back to Plaintext

**File:** `MerchantWallet.ts`

When encryption key env var is missing, encryption silently falls back to plaintext storage — no error, no warning, just stores credentials unencrypted.

---

### SCHEMA-009: Duplicate Indexes — 8 Models

Duplicate indexes found in: MerchantLoyaltyConfig, CancellationPolicy, Consultation, ServiceAppointment, StoreVisit, Order, Product, Store.

Duplicate indexes waste memory and slow down writes.

---

### SCHEMA-010: 19 Models Missing `createdAt` Index

Recency queries (e.g., "orders from last 7 days") scan entire collections without index on `createdAt`. Missing indexes identified across: Order, Payment, WalletTransaction, UserSession, Notification, StoreVisit, KarmaEarnRecord, KarmaRedeemRecord, LoyaltyTransaction, Offer, Coupon, Refund, Settlement, AuditLog, EventBooking, Review, SupportTicket, User, Merchant.
